# AI Usage Policy

ManaScope is maintained by humans. Every issue, discussion, and pull request is reviewed by a person. All contributions go through the same review process regardless of how they were written. To respect everyone's time, we ask that you put care and thought into your contributions.

We aren't anti-AI. In fact, we use it ourselves as a tool for refactoring, understanding code, and automating tedious tasks. However, to foster a good learning environment and prevent low-effort contributions, we have a few ground rules for using AI tools.

## The Rules

**1. Understand your code.**
Do not submit code entirely generated by AI ("vibe coded"). You must be able to explain how your code works and confidently make any requested changes.

**2. Test it yourself.**
Never submit untested code. Run the automated test suite *and* manually run the application to verify your changes work as expected. Do not use AI to generate code for platforms or environments you cannot test yourself.

**3. Discuss before you submit.**
AI-assisted pull requests should only address open issues. If you have a new idea, open an issue to get feedback before writing code. Drive-by PRs without an associated issue may be closed.

**4. Disclose your tools.**
If you use AI (e.g., Cursor, Claude, Copilot), mention it in your PR description. Let us know which tool you used and how it assisted you.

**5. Keep text concise.**
AI tends to be overly verbose. If you use it to help draft an issue or PR description, please review and edit it down to just the relevant facts.

**6. No AI media.**
AI assistance is permitted only for code and text. Do not submit AI-generated images, art, video, or audio. AI tools can also inadvertently reproduce copyrighted material in generated text or code, so please audit your contribution for any such content before submitting.

***

We genuinely enjoy helping developers learn and grow. That said, repeatedly submitting unverified, low-effort AI-generated code takes time away from everyone and may result in losing contribution privileges. If you're here to learn and collaborate, we're more than happy to guide you.

*(Inspired by the AI policies of [Ghostty](https://github.com/ghostty-org/ghostty/blob/main/AI_POLICY.md) and [KeePassXC](https://github.com/keepassxreboot/keepassxc?tab=readme-ov-file#generative-ai).)*
